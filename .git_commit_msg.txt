feat(dc): consolidate sellable UX into DC pages, revamp CRM settings

- sidebar: remove standalone /crm/sellable-potential nav (page kept for
  re-enable; all sellable data now lives on DC pages).
- dc_view: drop the CPU/RAM/Storage 'sold %' gauges from the DC summary
  CRM panel (compute, not sellable, is meant for those gauges); KPI cards
  now display Potential Revenue in shortened TL form (Milyon/Milyar/Trilyon)
  with a hover tooltip carrying the full grouped number.
- dc_view: add a 'Total Sellable Potential' KPI card at the top of both the
  Virtualization and Backup & Replication outer tabs, aggregating all virt
  / backup families respectively. Per-subtab sellable cards are kept for
  detail.
- _build_sellable_inline_kpi: TL sub-line now uses _fmt_tl_short with
  tooltip overlay so massive numbers stay legible without losing precision.
- datacenters list: 'Potansiyel gelir' summary KPI gets the same short
  format + explanatory tooltip describing the formula and zero-data caveat.
- CRM settings pages (panels, infra-sources, resource-ratios,
  unit-conversions, service-mapping): replace the static html.Tr tables
  with dash_table.DataTable carrying native filter/sort/row-select; click a
  row to load it into the editor form, Reset clears the form, Save
  refreshes the table in place. infra-sources additionally auto-fills the
  whole binding form when a panel is selected so operators edit existing
  rows instead of recreating them from blanks.
- service-mapping: instead of one inline Select+Save+Reset per product
  (unusable with 200+ rows), a single DataTable with native pagination/
  filter/sort plus an editor form above. UNMATCHED count surfaced as a
  badge in the summary strip.

Tests: tests/test_crm_settings_pages.py covers the new DataTable layouts
and the infra-source auto-fill helper. customer-api suite still 117/117.
